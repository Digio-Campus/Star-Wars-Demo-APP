# Prompt para verificar que los agentes de mesnada pueden acceder al contenedor docker con MacOS

Conecta por SSH y reporta: 1) Ejecuta: `ssh -p 50922 valero@localhost`. 2) Cuando se solicite contraseña, usa: deamu. 3) Captura todo el diálogo (stdout/stderr) y el código de salida. 4) Si la conexión SSH tiene éxito, sin hacer cambios destructivos, ejecuta y captura la salida de los siguientes comandos: `id`, `whoami`, `hostname`, `uname -a`, `ls -la ~`, `pwd`, `ps aux | head -n 20`, `which xcodebuild || echo xcodebuild_no_encontrado`, `ls /Applications || true`. 5) Comprueba si `sudo` funciona sin contraseña ejecutando `sudo -n true` y captura resultado (no intentes usar contraseña para sudo). 6) No realices cambios en el sistema ni ejecutes comandos que modifiquen archivos o instalen paquetes. 7) Si la conexión falla, intenta identificar la causa (timeout, rechazo de contraseña, conexión rechazada) y reporta cualquier mensaje de error. 8) Al final, sintetiza un breve resumen: si la SSH fue posible, qué privilegios tiene el usuario, qué herramientas de desarrollo (Xcode/xcodebuild) están disponibles, y qué pasos recomendarías al usuario para continuar. Entrega: transcript completo + resumen final claro y conciso. Nota: Las credenciales y el host son proporcionados por el solicitante; procede sólo a comprobación remota y reporta. No compartas la contraseña en el informe salvo en la trasncripción necesaria para reproducir localmente.
